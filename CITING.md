# Citing Salesforce AgentMemory

If you use AgentMemory in your research, architecture documentation, technical articles, Salesforce implementations, or presentations, please cite this project to help others discover it.

---

## GitHub Citation Button

The easiest way to cite this project is to use GitHub's built-in citation feature. Click the **"Cite this repository"** button in the right sidebar of the repository page. GitHub will generate a citation in both APA and BibTeX formats from the `CITATION.cff` file.

---

## Citation Formats

### BibTeX (for LaTeX / academic papers)

```bibtex
@software{raju2026agentmemory,
  author       = {Raju, Jayavardhan},
  title        = {{Salesforce AgentMemory: Cross-Cloud Contextual Memory Framework}},
  year         = {2026},
  url          = {https://github.com/jayavardhan-raju/Salesforce-AgentMemory},
  version      = {1.0.0},
  note         = {Platform-native Apex framework with self-tuning confidence engine
                  for cross-cloud intelligence across Sales, Service, and Marketing Cloud}
}
```

### APA (7th Edition)

> Raju, J. (2026). *Salesforce AgentMemory: Cross-Cloud Contextual Memory Framework* (Version 1.0.0) [Computer software]. https://github.com/jayavardhan-raju/Salesforce-AgentMemory

### IEEE

> J. Raju, "Salesforce AgentMemory: Cross-Cloud Contextual Memory Framework," 2026. [Online]. Available: https://github.com/jayavardhan-raju/Salesforce-AgentMemory

### Chicago (Author-Date)

> Raju, Jayavardhan. 2026. "Salesforce AgentMemory: Cross-Cloud Contextual Memory Framework." https://github.com/jayavardhan-raju/Salesforce-AgentMemory.

### Plain Text / Blog / Presentation Attribution

> Salesforce AgentMemory by Jayavardhan Raju — https://github.com/jayavardhan-raju/Salesforce-AgentMemory

## Why Cite?

Citing open-source projects helps the author demonstrate impact and community adoption. It also helps other architects and developers discover frameworks that solve cross-cloud intelligence problems within the Salesforce platform.

If you've used AgentMemory in a production org, conference talk, blog post, or technical design — a citation or a GitHub star goes a long way.
